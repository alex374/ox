# AI 设计师助手 - 使用指南

这份指南将帮助您快速上手并充分利用 AI 设计师助手的各项功能。

## 🚀 快速开始

### 1. 环境配置

首先确保您已正确配置环境变量：

### 2. 配置 API 密钥

复制环境变量配置文件：

```bash
cp .env.example .env
```

在 `.env` 文件中配置以下密钥：

```bash
# OpenRouter API Key (必需 - 用于聊天功能)
VITE_OPENROUTER_API_KEY=your_openrouter_api_key_here

# OpenAI API Key (可选 - 用于 DALL-E 图片生成)
VITE_OPENAI_API_KEY=your_openai_api_key_here
```

**重要说明**：
- OpenRouter API 密钥是必需的，用于基本的聊天功能
- OpenAI API 密钥是可选的，用于真实的图片生成功能
- 如果没有提供 OpenAI API 密钥，系统会使用高质量的模拟图片作为备选方案

### 3. 启动应用

```bash
npm install
npm run dev
```

应用将在 `http://localhost:5173` 启动。

## 📝 使用方法

### 基本聊天功能

配置完成后，您可以直接开始使用：

1. 在聊天输入框中描述您的设计需求
2. AI 助手会理解您的需求并提供专业建议
3. 如果涉及具体设计，系统会自动生成设计稿

当您的需求涉及具体的界面设计时，系统会：

1. **智能识别**: 自动识别是否需要生成设计稿
2. **图片生成**: 
   - 优先使用 OpenAI DALL-E 生成真实设计稿（需要 OpenAI API 密钥）
   - 备选使用高质量模拟图片
3. **画廊展示**: 在右侧设计稿画廊中展示所有生成的设计稿

## 🔧 配置说明

### OpenRouter API 配置

OpenRouter 提供多模型 API 接入，支持 GPT-4 等先进模型：

1. **注册账户**: 访问 [OpenRouter](https://openrouter.ai)
2. **获取 API 密钥**: 在控制台生成新的 API 密钥
3. **配置额度**: 确保账户有足够的 API 调用额度
4. **设置环境变量**: 将密钥添加到 `.env` 文件

### OpenAI API 配置（可选）

用于 DALL-E 图片生成功能：

1. **注册账户**: 访问 [OpenAI Platform](https://platform.openai.com)
2. **获取 API 密钥**: 在 API Keys 页面创建新密钥
3. **配置额度**: 确保账户有足够的图片生成额度
4. **设置环境变量**: 将密钥添加到 `.env` 文件

## 🎨 设计理念

### UI/UX 原则

AI 助手基于以下设计原则提供建议：

- **简洁性**: 追求简洁明了的界面设计
- **一致性**: 保持设计元素的一致性
- **可用性**: 优先考虑用户体验
- **美观性**: 注重视觉美感和现代感
- **响应式**: 适配不同设备和屏幕尺寸

### 设计流程

1. **需求分析**: 理解用户的具体需求
2. **方案建议**: 提供多种设计方案
3. **细节完善**: 深入讨论设计细节
4. **视觉呈现**: 生成具体的设计稿

## 🛠️ 故障排除

### 1. API 密钥错误

**问题**: "OpenRouter API key is required" 或 "Failed to communicate with AI service"

**解决方案**:
- 检查 `.env` 文件中的 API 密钥配置
- 确认 API 密钥格式正确（以 `sk-or-v1-` 开头）
- 确认 API 密钥有效且有足够余额

### 2. 图片生成失败

**问题**: 设计稿显示为模拟图片

**解决方案**:
- 检查 OpenAI API 密钥是否正确配置
- 确认 OpenAI 账户有足够的图片生成额度
- 如果不需要真实图片生成，模拟图片也能满足基本需求

### 3. 网络连接问题

**问题**: 请求超时或连接失败

**解决方案**:
- 检查网络连接状态
- 确认防火墙没有阻止相关请求
- 尝试刷新页面重新连接

### 4. 页面加载异常

**问题**: 页面显示异常或功能无法使用

**解决方案**:
- 检查浏览器控制台错误信息
- 清除浏览器缓存
- 确认所有依赖都已正确安装
